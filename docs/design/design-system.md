# SCBA Manager — Дизайн-система (MVP)

> Версія 1.0 від 2026-07-10. Автор: designer-агент.
> Джерело вимог: `docs/requirements.md` (ЗАТВЕРДЖЕНО). Закриває задачу **D-1** (розділ 10).
> Мова інтерфейсу — українська.

---

## 1. Принципи

1. **Рукавиці перш за все.** Мінімальний тач-таргет 48×48 px; основні дії — 56+ px;
   Старт/Стоп заправки — 96 px на всю ширину. Відстань між сусідніми таргетами ≥ 8 px.
2. **Статус = колір + іконка + текст.** Жоден статус не передається лише кольором
   (WCAG 1.4.1). Контраст тексту ≥ 4.5:1 (WCAG AA), великих елементів ≥ 3:1.
3. **Читабельність на сонці / у поганому освітленні.** Світла тема з високим контрастом,
   базовий текст #101828 на білому (контраст ~17:1). Темна тема — поза MVP.
4. **Мінімум кроків до критичного.** Прострочене обладнання — перше, що видно після
   входу (мейнборд). Заправка — 1 тап із будь-якого екрана (майстер).
5. **Безпекова точність.** Дати й наробіток завжди показуємо явно (число + одиниця),
   не лише «через N днів».

| `color-text-inverse` | `#FFFFFF` | Текст на темних/кольорових фонах |
| `color-bg-page` | `#F8FAFC` | Фон сторінки |
| `color-bg-surface` | `#FFFFFF` | Картки, панелі, модалки |
| `color-border` | `#D0D5DD` | Межі полів, розділювачі |
| `color-brand` | `#1D4ED8` | Первинні кнопки, активна вкладка, лінки |
| `color-brand-hover` | `#1E40AF` | Hover/pressed первинних кнопок |

Контраст: `#1D4ED8` на білому — 6.3:1 (AA для тексту будь-якого розміру).

### 2.2 Статусні кольори (ядро D-1)

Кожен статус має **три складові**: заливка (bg), текст/іконка (fg), акцентна смуга (accent).
Текст статусу завжди темний на світлому фоні — це стабільно читається на сонці.

| Токен | bg | fg (текст + іконка) | accent (смуга/бордер) | Контраст fg/bg |
|---|---|---|---|---|
| `status-danger` (Прострочено / Несправний) | `#FEF3F2` | `#B42318` | `#D92D20` | 6.6:1 |
| `status-warning` (Увага / Наближається) | `#FEF0C7` | `#93370D` | `#DC6803` | 7.5:1 |
| `status-ok` (Справний / У нормі) | `#ECFDF3` | `#067647` | `#12B76A` | 5.9:1 |
| `status-neutral` (У ремонті / Розібраний) | `#F2F4F7` | `#475467` | `#98A2B3` | 7.9:1 |
| `status-archived` (Списаний / Архів) | `#F2F4F7` | `#667085` + strikethrough назви | `#D0D5DD` | 5.6:1 |

Суцільні заливки (для лічильників-«чипів» на мейнборді, де треба «кричати»):

| Токен | bg | текст | Контраст |
|---|---|---|---|
| `status-danger-solid` | `#B42318` | `#FFFFFF` | 6.6:1 |
| `status-warning-solid` | `#93370D` | `#FFFFFF` | 7.5:1 |

> Важливо: класичний «жовтий на білому» не проходить AA, тому для суцільного
> варіанта попередження використовуємо темно-бурштиновий `#93370D`, а «жовтизна»
> передається світлою заливкою `#FEF0C7` у badge/рядках.

### 2.3 Іконки статусів (обов'язкові, поруч із текстом)

| Статус | Іконка (Lucide) | Форма (дублює зміст для дальтоніків) |
|---|---|---|
| Прострочено / Несправний | `octagon-alert` (восьмикутник, як знак STOP) | восьмикутник |
| Увага | `triangle-alert` (трикутник із «!») | трикутник |
| У нормі | `check-circle` | коло з галочкою |
| У ремонті | `wrench` | ключ |
| Розібраний (апарат без балонів) | `unlink` / `puzzle` | розірваний ланцюг |
| Списаний / Архів | `archive` | коробка |
| Гідротест | `droplets` | краплі |
| Редуктор | `gauge` | манометр |
| ТО компресора | `settings` / `hard-hat` | шестерня |

Різні **форми** іконок (восьмикутник / трикутник / коло) — гарантія розрізнення
без кольору.

## 3. Система статусної індикації (D-1)

### 3.1 Правила обчислення статусів (для UI; розрахунок — на backend)

**Балон** — найгірший із:
| Умова | Статус | Текст badge |
|---|---|---|
| Гідротест прострочено (`next_hydro_date < today`) | Прострочено | `Гідротест прострочено · 12.06.2026` |
| До гідротесту ≤ **30 днів** | Увага | `Гідротест через 18 дн · 28.07.2026` |
| Кінець строку служби минув | Прострочено | `Строк служби вичерпано · 01.05.2026` |
| До кінця строку служби ≤ **90 днів** (пропозиція, див. ВП-1) | Увага | `Строк служби до 05.10.2026` |
| Інакше | У нормі | `Гідротест до 14.03.2027` |

**Ложамент** — найгірший із:
| Умова | Статус | Текст badge |
|---|---|---|
| Заміна редуктора прострочена | Прострочено | `Редуктор прострочено · 02.07.2026` |
| До заміни редуктора ≤ **60 днів** (дефолт для «1–2 міс», див. ВП-2) | Увага | `Редуктор через 45 дн · 24.08.2026` |
| Статус «у ремонті» | У ремонті | `У ремонті` |
| Статус «списаний» | Архів | `Списаний` |
| Інакше | У нормі | `Редуктор до 24.08.2027` |

**Апарат** = найгірший статус компонентів (успадкування):
| Умова | Статус | Текст badge |
|---|---|---|
| Будь-який балон або ложамент прострочені | **Несправний** (червоний) | `НЕСПРАВНИЙ · гідротест балона №0417` |
| Будь-який компонент «жовтий» | Увага | `Увага · редуктор через 45 дн` |
| Немає балонів | Розібраний (нейтральний) | `Розібраний · без балонів` |
| Інакше | Справний | `Справний` |

Текст badge апарата завжди називає **причину** (який компонент і чому) — майстру
не треба провалюватись у картку, щоб зрозуміти проблему.

**Компресор** — найгірший із:
| Умова | Статус | Текст badge |
|---|---|---|
| Наробіток ≥ порога наступного ТО, або календарний строк ТО-1000/2000 минув | Прострочено | `ТО-125 прострочено · +6.2 мг` |
| До наступного ТО ≤ **5 мг** або ≤ **30 днів** до календарного ТО-1000/2000 (пропозиція, див. ВП-3) | Увага | `ТО-25 через 3.5 мг` |
| Інакше | У нормі | `ТО-25 через 14.2 мг` |

**Пріоритет при агрегації (сортування, підрахунок):**
`Прострочено (0) → Увага (1) → У ремонті/Розібраний (2) → У нормі (3) → Архів (4)`.
Архівні записи не потрапляють у мейнборд і лічильники; доступні через фільтр «Показати списані».

### 3.2 Як статус виглядає в UI (три рівні індикації)

**Рівень 1 — рядок списку / картка (`ListRow`, `EntityCard`):**
```
┌──────────────────────────────────────────────┐
│▌ bS-4343234            [⬣ Гідротест простр.] │   ▌ = смуга 6 px кольору accent
│▌ Drager PSS 4000 · 2 балони    12.06.2026    │       уздовж лівого краю
└──────────────────────────────────────────────┘
```
- Ліва смуга 6 px (`status-*` accent) — видно здалеку, «сканується» оком по списку.
- Badge (іконка + текст) у правому верхньому куті рядка.
- Для «Прострочено» фон усього рядка — `status-danger.bg` (#FEF3F2), для «Увага» —
  фон білий, лише смуга + badge (щоб червоне домінувало візуально).

**Рівень 2 — badge (`StatusBadge`):**
- Висота 32 px (у рядках) / 40 px (у шапці картки деталей).
- Радіус 8 px, іконка 20 px + текст 14–16 px semibold, padding 6–10 px.
- Ніколи не «тільки крапка»: завжди іконка + слово.

**Рівень 3 — лічильники на мейнборді (`CounterChip`):**
- Суцільна заливка `status-danger-solid` / `status-warning-solid`, білий текст.
- Висота 56 px, це одночасно **фільтри-кнопки** (тап — фільтрує список нижче).
```
[ ⬣ Прострочено: 3 ]  [ ⚠ Увага: 7 ]  [ ✓ У нормі: 41 ]
```

**Дублювання в навігації:** пункт «Головна» в сайдбарі/нижньому меню носить червону
цятку-лічильник (badge з числом), якщо є прострочене, — але число + форма, не лише колір.

### 3.3 Рукавиці та мобільний екран

- Увесь рядок списку — тач-таргет (мін. 64 px на мобільному, 56 px на ПК). Не окремі дрібні іконки.
- Badge не є кнопкою — дії лише через рядок/великі кнопки (виключаємо промахи).
- Свайпи не використовуємо (в рукавицях ненадійні); лише тапи.
- Текст статусу ≥ 14 px semibold; дати — tabular numerals.

Базовий крок — **4 px**: `space-1=4, space-2=8, space-3=12, space-4=16, space-5=20, space-6=24, space-8=32, space-10=40, space-12=48`.

## 6. Компоненти та їх стани

### 6.1 `StatusBadge`
Пропси: `status` (`danger|warning|ok|neutral|archived`), `icon`, `label`, `size` (`sm|md`).
Станів loading/error не має — чистий презентаційний.

### 6.2 `ListRow` / `EntityCard`
- **default** — смуга статусу + назва + мета + badge; увесь рядок клікабельний.
- **pressed/hover** — фон `#F2F4F7`; фокус — ring 2 px.
- **selected** (у виборі компонентів апарата, у сесії заправки) — бордер 2 px `color-brand`, чекмарк 24 px праворуч.
- **disabled** (компонент зайнятий іншим апаратом) — opacity .5 + підпис причини: `В апараті bS-118`. Не приховуємо — пояснюємо.

### 6.3 Списки (`EntityListPage`)
- **loading** — 6 скелетон-рядків (пульсація), висота = `touch-row`.
- **empty** — іконка сутності 48 px, заголовок «Балонів поки немає», кнопка `+ Додати балон` (для майстра; черговому — лише текст).
- **empty за фільтром** — «Нічого не знайдено за фільтром», кнопка «Скинути фільтри».
- **error** — іконка `cloud-off`, «Не вдалося завантажити. Перевірте з'єднання», кнопка «Повторити» (48 px).

### 6.4 Кнопки (`Button`)
Варіанти: `primary` (brand, білий текст), `secondary` (біла, бордер), `danger` (danger-solid), `ghost`.
Стани: default / hover / pressed / **loading** (спінер 20 px + текст лишається, кнопка заблокована) / disabled (opacity .5).
Розмір за замовчуванням — `touch-button`; `size="xl"` = `touch-cta`.

### 6.5 `BigActionButton` (Старт/Стоп сесії)
- 96 px, ширина 100%, текст 24 px + іконка 32 px.
- «СТАРТ» — суцільний зелений `#067647` (білий текст, 5.9:1), іконка `play`.
- «СТОП» — суцільний червоний `#B42318`, іконка `square`. Стоп вимагає **утримання 1 с**
  (кільце прогресу навколо іконки) або підтвердження — захист від випадкового тапу рукавицею.
- loading (збереження сесії) — спінер, кнопка заблокована, текст «Зберігаю…».

### 6.6 Поля форм (`FormField`, `SelectField`, `DateField`, `NumberStepper`)
- Висота 56 px, label зверху (не placeholder-only), 16–18 px.
- **error** — бордер `#D92D20` 2 px + текст помилки під полем із іконкою `alert-circle` (не лише колір).
- `NumberStepper` (тиски, інтервали): поле + кнопки `−`/`+` по 48 px — зручно в рукавицях, клавіатура не обов'язкова. Для тиску кроки по 5 бар, швидкі пресети `200`, `300` бар.
- `DateField`: нативний date-picker + ручне введення `дд.мм.рррр`.

### 6.7 `SelectSheet` (вибір ложамента/балона/компресора/місця зберігання)
На мобільному — bottom sheet на повний екран зі списком-`ListRow` + пошук зверху;
на ПК — модалка 560 px. Ніяких дрібних дропдаунів для довгих списків.

### 6.8 `QRScannerSheet`
Повноекранна камера, рамка наведення, підпис «Наведіть на QR-код ложамента».
Стани: **scanning** / **found** (зелений спалах рамки + вібрація + звук, показ `bS-…`, авто-додавання) /
**not-found** («Ложамент bS-… не знайдено на станції», кнопка «Сканувати ще») /
**no-permission** («Немає доступу до камери», кнопка «Ввести номер вручну»).
Fallback ручного введення завжди на екрані.

### 6.9 `ProgressToMaintenance` (прогрес до ТО)
Лінійний прогрес-бар 12 px: `наробіток від останнього ТО / інтервал`.
Кольори: <80% — accent ok; 80–100% — warning; >100% — danger + текст `+N мг понад інтервал`.
Поруч завжди текст: `18.6 / 25 мг до ТО-25` (+ для 1000/2000: `або до 14.03.2027`).

### 6.10 `HistoryTimeline`
Вертикальний список подій (гідротести, заміни балонів, сесії заправки, ТО):
дата — жирна, тип події — іконка + назва, деталі — secondary. Стани loading (3 скелетони) / empty («Історія порожня»).

### 6.11 `Toast` / `ConfirmDialog`
- Toast: успіх (ok-палітра) / помилка (danger), 4 с, зверху по центру, ≥ 48 px, кнопка закриття 48 px.
- ConfirmDialog для незворотних дій (списання, «Провести ТО», Стоп сесії): заголовок-питання, наслідок текстом, кнопки 56 px, небезпечна — danger праворуч.

### 6.12 `RoleGate` (черговий = viewer)
Для ролі «черговий» всі кнопки створення/редагування **не рендеряться** (не disabled —
прибрані), форми відкриваються в read-only. У хедері — badge ролі `Перегляд`.

### 6.13 Банер з'єднання (`ConnectionBanner`)
Офлайн — скіп у MVP, але при втраті мережі показуємо липкий банер зверху:
`⚠ Немає з'єднання — дані можуть бути неактуальні` (warning-палітра). Без черги синхронізації.

## 7. Іконографія та інші правила

- Набір: **Lucide** (MIT, є в реакт-пакеті `lucide-react`). Розмір 20/24/32 px, stroke 2.
- Дати завжди у форматі `дд.мм.рррр`; відносний час дублюється абсолютним: «через 18 дн · 28.07.2026».
- Мотогодини: `мг`, один знак після коми (`18.6 мг`). Тиск: `бар`, ціле число.
- a11y: усі статуси мають `aria-label` із повним текстом; фокус-обведення видиме;
  таймер сесії — `aria-live="off"` (не спамити скрінрідер), кнопки Старт/Стоп — чіткі `aria-label`.

## 8. Відкриті питання дизайн-системи (для PM)

- **ВП-1:** поріг «жовтого» для кінця строку служби балона — пропоную 90 днів. Підтвердити.
- **ВП-2:** «1–2 місяці» для редуктора — пропоную дефолт **60 днів** без налаштування в MVP. Підтвердити.
- **ВП-3:** поріг «жовтого» для ТО компресора — пропоную **≤ 5 мг** до порога (і ≤ 30 днів для календарних ТО-1000/2000). Підтвердити із власником.
