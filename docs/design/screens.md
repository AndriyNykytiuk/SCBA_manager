## 0. Мапа екранів і навігація

```
/login
/                     Головна (мейнборд прострочень/наближень) — стартова після входу
/backplates           Ложаменти: список → /backplates/:id (картка) → форма
/cylinders            Балони:    список → /cylinders/:id → форма
/apparatus            Апарати:   список → /apparatus/:id → форма-збірка
/compressors          Компресори: список → /compressors/:id (картка + історія + «Провести ТО»)
/fill-session         Флоу сесії заправки (майстер): 3 кроки + екран таймера
/admin/users          Admin: користувачі
```

**Навігація (лівий дашборд §3.6):** Головна · Ложаменти · Балони · Апарати ·
Компресори (+ Користувачі — лише admin). Кнопка **«Заправка»** — окрема
акцентна CTA (не вкладка), бо це найчастіша дія майстра.

**Ролі:** черговий бачить усе те саме без кнопок дій (`RoleGate`);
admin додатково має перемикач станції в хедері.

Правила мейнборда:
- Сортування: спочатку всі червоні, потім жовті; всередині групи — за датою/наробітком (найближче/найпростроченіше вгорі).
- Кожен рядок називає **сутність, причину і дату**; тап → картка сутності.
- Якщо прострочень немає: секція «⬣ Прострочено» не рендериться; якщо взагалі чисто —
  empty-стан: `✓ Все обладнання в нормі` (велика зелена іконка + текст) + чип «У нормі: N».
- Loading: 3 скелетон-чипи + 6 скелетон-рядків. Error: стан з «Повторити» (див. ДС §6.3).

### 4.2 Ложамент `/backplates/:id`

наступна дата + статус, кнопка «Зафіксувати заміну редуктора»), секція «Дані»
(виробник/модель/серійник/дата вводу/примітки/статус), зв'язок «В апараті → …».

«у ремонті/списаний» ложамента, що стоїть в апараті, — блокується з поясненням
«Спочатку розберіть апарат bS-…».

## 5. Форми створення/редагування

### 5.1 Спільні правила
- Одна колонка, поля 56 px, label зверху; на ПК форма в модалці/панелі 560 px, на мобільному — окремий екран.
- Кнопки внизу: «Скасувати» (secondary) + «Зберегти» (primary, loading-стан).
- Валідація по blur + при сабміті; помилки — біля полів (ДС §6.6).

### 5.2 Балон — створення
Поля: номер* · об'єм (сегмент `6 / 6.8 / 7 л`) · матеріал (сегмент `Метал / Композит`) ·
робочий тиск (`NumberStepper`, пресет 300) · виробник · дата виготовлення* ·
кінець строку служби* · дата останнього гідротесту · **інтервал гідротесту***
(число + одиниця міс/роки; підказка «для металу й композиту інтервали різні») →
під полем живе прев'ю: `Наступний гідротест: 12.06.2031` + лінк «Змінити вручну»
(розблоковує поле дати).

### 5.3 Ложамент — створення
Назва/номер* (формат bS-…, валідація не жорстка — лише не порожнє) · виробник ·
модель · серійний номер · дата введення в експлуатацію · дата останньої заміни
редуктора · інтервал заміни редуктора* (міс/роки) → прев'ю наступної заміни ·
примітки · статус.

### 5.4 Апарат — збірка (ключова форма)

```
Крок єдиний (одна форма), але вибори — через SelectSheet:

┌ Новий апарат ──────────────────────────────┐
│ 1. Ложамент *                              │
│ ┌ [ Обрати ложамент… ]────────────────── ┐ │ → SelectSheet: лише «вільні»
│ └─────────────────────────────────────── ┘ │   ложаменти; зайняті видно
│    Апарат отримає номер ложамента          │   сірими з підписом «В апара-
│                                            │   ті bS-118» (disabled)
│ 2. Балони (0–2)                            │
│ ┌▌🛢 №0533   [⚠ 18 дн]      [Прибрати 48]┐ │
│ └────────────────────────────────────────┘ │
│ [ + Додати балон ]                         │ → SelectSheet: лише вільні
│    Без балонів апарат буде «розібраним»    │   балони; макс. 2
│                                            │
│ 3. Місце зберігання                        │
│ [ Обрати місце… ▾ ]                        │ → довідник storage_location
│                                            │
│ [ Скасувати ]              [ Створити ]    │
└────────────────────────────────────────────┘
```
- Додавання «жовтого» балона дозволене (з badge видно статус), **червоного** — теж
  дозволене, але з попередженням: «Апарат одразу стане несправним» (ВП-5).
- Редагування апарата = той самий екран; заміна балона автоматично пише запис в
  «Історію замін балонів».

### 5.5 Компресор — створення
Назва/модель* · **поточні мотогодини*** · **дата останнього ТО*** · **наробіток на
момент останнього ТО*** (+ рівень останнього ТО — сегмент). Більше нічого (§3.4).

## 6. Флоу сесії заправки `/fill-session` (§3.5)

Доступ: лише майстер. Вхідні точки: CTA «⛽ Заправка» в хедері/таб-барі, кнопка
«Заправка» на картці компресора (тоді крок 1 пропускається — компресор передвибраний).

**Степер: `1 Компресор → 2 Апарати → 3 Тиски → Таймер`.** Прогрес зверху
(«Крок 2 з 3»), «Назад» доступний до Старту.

### Крок 1 — Компресор
Список компресорів станції (`ListRow` + наробіток + статус ТО). Якщо компресор
«⬣ ТО прострочено» — вибір дозволений, але з попередженням-рядком
`⚠ У компресора прострочене ТО-125` (не блокуємо роботу частини).

### Крок 2 — Що заправляємо

```
┌ Крок 2 з 3 · Апарати та балони ────────────┐
│ ┌──────────────────┐ ┌──────────────────┐  │
│ │  📷 Сканувати QR │ │ ⌨ Ввести номер   │  │ 72px обидві
│ └──────────────────┘ └──────────────────┘  │
│ [ + Окремий балон (без апарата) ]  48px    │ → SelectSheet вільних балонів
│                                            │
│ Додано (3):                                │
│ ┌▌🧰 bS-118        [✓]        [✕ 48px] ┐   │
│ ┌▌🧰 bS-4343234    [⬣ НЕСПРАВНИЙ] [✕] ┐    │ ← додати можна, попередження
│ ┌▌🛢 №0912 (окремий балон) [✓] [✕] ┐       │
│                                            │
│ [ Далі: тиски → ]            72px primary  │
└────────────────────────────────────────────┘
```
- QR-скан (ДС §6.8): успішний скан додає апарат одразу, камера лишається відкритою
  для серійного сканування (лічильник «Додано: 3»), вібро+звук на кожен скан.
- Ручне введення: поле + автопідказки за номером ложамента.
- Дублікати не додаються (тост «bS-118 вже в сесії»).
- «Далі» неактивна, поки список порожній.

### Крок 3 — Тиски (один раз на всю сесію, U-5)

```
┌ Крок 3 з 3 · Тиски на групу ───────────────┐
│ Тиск до заправки, бар                      │
│   [−]      180      [+]     (степпер, крок │
│ Цільовий тиск, бар            5, поле теж  │
│   [−]      300      [+]      редагується)  │
│   Пресети: [200] [300]                     │
│                                            │
│ Компресор: Bauer K-14 · Апаратів: 2 · +1 б.│
│ ┌────────────────────────────────────────┐ │
│ │            ▶  СТАРТ                    │ │ 96px зелена
│ └────────────────────────────────────────┘ │
└────────────────────────────────────────────┘
```
Валідація: цільовий > тиску до заправки; обидва > 0 (помилка текстом під полем).

### Екран таймера (сесія триває)

```
┌ Іде заправка · Bauer K-14 ─────────────────┐
│                                            │
│              00:14:37                      │  font-timer, tabular
│         почато о 14:02                     │
│                                            │
│   180 → 300 бар      Апаратів: 2 · +1 бал. │  font-kpi / caption
│   [ Показати список ▾ ]                    │
│                                            │
│ ┌────────────────────────────────────────┐ │
│ │        ■  СТОП (утримуйте)             │ │ 96px червона,
│ └────────────────────────────────────────┘ │ hold 1с (ДС §6.5)
└────────────────────────────────────────────┘
```
- Таймер живе на сервері (старт зафіксований) — закриття вкладки/додатка не губить
  сесію: при поверненні майстра застосунок показує банер «Іде заправка · 00:41:12 ·
  [Відкрити]» поверх усіх екранів. Забутий «Стоп» — без автокорекції (U-5).
- Після Стопу — екран підсумку: тривалість, `+0.4 мг` до наробітку, новий прогрес
  до ТО (`ProgressToMaintenance`), список заправленого; кнопки «Готово» (→ Головна)
  і «Нова сесія» (→ Крок 2 з тим самим компресором).
- Error при збереженні: «Сесію не збережено. [Повторити]» — дані сесії не втрачаються.

## 7. Admin: станції та користувачі

- **Перемикач станції** — в хедері: `[Станція: ДПРЧ-12 ▾]` → SelectSheet зі списком
  станцій (рядки 64 px, у кожної — міні-лічильники `⬣ N ⚠ N`, щоб admin одразу бачив
  проблемні частини). Вибір станції перемикає ВЕСЬ застосунок у контекст цієї станції
  (мейнборд, списки, заправка). Поточна станція завжди видима в хедері.
- **/admin/users** — список користувачів: ПІБ, логін, роль (badge: Admin/Майстер/
  Черговий), станція, статус (активний/деактивований). Кнопка «+ Користувач» →
  форма: ПІБ, логін, пароль, роль (сегмент), станція (select; для admin —
  недоступний, глобальний). Дії: «Скинути пароль», «Деактивувати» (ConfirmDialog).
- Майстер/черговий пункт «Користувачі» і перемикач станції не бачать.

## 9. Відкриті питання (PM / власник)

- **ВП-4 (PM):** склад мобільного таб-бару — пропоную Головна · Апарати · Заправка ·
  Балони · Ще (Ложаменти/Компресори в «Ще»). Підтвердити пріоритет вкладок у польових умовах.
- **ВП-5 (PM):** чи дозволяти збирати апарат із простроченим балоном (зараз —
  дозволено з попередженням, апарат одразу несправний)? Альтернатива — блокувати.
- **ВП-6 (backend):** «активна сесія заправки» має жити на сервері (старт-час),
  щоб таймер переживав перезавантаження сторінки і був видимий з інших пристроїв.
  Узгодити з B-1 (endpoint активної сесії).
- **ВП-7 (PM/backend):** друк QR-кодів (QrCodeView) — чи потрібен у MVP, чи QR
  наносяться зовнішнім способом? Зараз закладено лише перегляд/друк з картки апарата.
- ВП-1…ВП-3 (пороги «жовтого») — див. `design-system.md` §8.
